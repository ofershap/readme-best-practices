Create or improve the README.md for this project using the readme-best-practices skill.

Steps:

1. Read the project source to understand what it does and who it's for
2. Check for existing assets (demo GIF, screenshots, logo) in `assets/`
3. Look at other READMEs in the workspace for established conventions
4. If a README exists, identify what can be improved. If not, create one from scratch
5. Apply the full readme-best-practices skill structure:
   - BLUF header with bold punchline
   - Navigation buttons (for-the-badge style)
   - Status badges (only truthful ones)
   - Story section with a personality-driven name
   - Demo GIF (only if asset exists)
   - Copy-paste Quick Start
   - Feature tables (not bullet lists)
   - Architecture / tech stack
   - Author section with GitShow card
   - License
6. Verify no anti-patterns remain (em dashes, AI vocabulary, broken asset links, generic section
   names)
